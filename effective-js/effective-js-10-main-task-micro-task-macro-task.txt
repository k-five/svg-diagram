میدونم که جاواسکریپت تک نخی هست
اما اینکه چطور ایسنک اپریشن ها رو هندل میکنه
asynchronous operations
به این صورت هست که اونها رو اولیت بندی میکنه

کدی که اولیت اجرا بالاتری داره رو الان انجام میده
کدی که ممکنه باعث تاخیر در نخ اصلی بشه رو جدا میکنه
یعنی همون لحظه اجرا نمیکنه
مثلا
console.log( "js" );
رو الان اجرا میکنه ولی
setTimeout( callback, time );
رو میبره میزاره توی ایونت کیو
Event Queue
تا کی؟

تا وقتی که ستک جایی که کد ها اجرا میشن خالی باشه
وقتی خالی باشه فرصت اجرا پیدا میکنه

نکته این نوع از اجرا کد اینجاست اگر برنامه نویس کدی رو اجرا کنه
که نخ اصلی رو خیلی درگیر کنه و یا کامل درگیر کنه هیچ ایونت دیگه ایه فرصت اجرا پیدا نمیکنه
حتا رندر شد صفحه چون اولویت پایی داره

مثال معمولش اینکه که روی سکرول صفحه لیسینر بذاریم
on scroll
وقتی سکرول میکنیم ایونت ها سکرول کجا میرن؟ توی ایونت کیو
حالا اگه با جاواسکریپت یعنی نخ اصلی انیمیشن سنگینی رو اجرا کنیم چی میشه؟
خیلی ساده س
ایوینت کیو پر میشه و ایونت ها یا اجرا نمیشن یا خیلی کند میشن چون ستک اجرا کد خالی نیست

اگه جاواسکریپت
parallel execution
داشت مشکلی نداشتیم ولی نداره

event queue
event loop
parallel execution
