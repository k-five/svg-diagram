اون قسمتی از او او پی رو که خودم جزیات رو کد میزنیم حالتی از
imperative API
هست و استفاده از این اینترفیس که نوشتیم یا دیگران نوشتن حالتی از
declarative API
هست

در اف پی این جزیات قبلا کد زده شدن
نه به صورت رندم یا خواست یه نفر یا گروه خاص
بکله بر اساس قواعد از پیش ثابت شده ریاضی

ریاکت مخصوصا جی اس اکس هم همین داستانه
بر اساس پارسر اکس ام ال فانکشن های نیتیو دام رو رپ کردین
JSX (= JavaScript Syntax Extension)
e.g.
const h1 = <h1>This is h1 tag</h1>
https://reactjs.org/docs/introducing-jsx.html

در این ستایل جزیات از ما مخفی میشه و ما از ابسترکشن بالاتری استفاده میکنیم
نکته بدش اینجاست که باید این ابسترکشن بلاتر رو یاد بگیریم
نکته خوبش اینجاست وقتی یاد گرفتیم جبران میشه برامون

in imperative style
ما معمولا ستیت داریم
معمولا لوپ و برنچ - یعنی ایف - داریم
اگه مواضب نباشیم به گاد کلاس میریسم
God Class
و دیباگ کردن سخت تری داریم
تست نویسی هم سخت تره

in declarative style
ستیت هاموم تا جایی که میشه محدود میکنیم
تقریبا لوپ و برنچی نمی بینیم
کلاسی در کار نیست و اگرم هست خیلی کم سایز هست
دیباگ کردن راحتره
تست نویسی هم راحت تره

واقعا اف پی انقد خوبه؟
پس چرا او او پی مشهور تره و بیشتر استفاده میشه؟

جواب این دو تا سوال رو در پست بعدی متوجه میشیم
بریم برای پست بعدی
