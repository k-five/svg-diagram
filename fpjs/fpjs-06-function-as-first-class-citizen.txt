اولین و مهمترین مفهنوم؟ خود فانکشن هست
ساده ترین تعریف اون؟
بلاکی از کد که قابل اجراست با اپوریتورهای پرانتز
()

این فیچر در همه زبان ها هست
اما در اون های که به طور خاص اف پی هستن
یا تا حدودی اون رو ساپورت میکنن یه فیچر دیگه هم هست
که میتونیم یه فانکشن رو در ردیف یه داده قرار بدیم

اون رو به یه متغیر اساین کنیم
اون رو پاس بدیم و یا برگردونیم
اون رو در یه ارایه ذخیره کنیم

سختی کار دونستن این موضوع نیست
سختی کار عادت دادن ذهن به این موضوع است
و بهترین تمرین برای ذهن هم این هست که
فانکشن رو اسایت کنیم یعنی از
function expression
استفاده کنیم تا عادت کنیم مثل
const add = function( a, b ){ ... }
و یا
const add = ( a, b ) => a + b;

حالا مهمترین چیزی که این فیچر بهمون میده
این نیست که میتونیم اون رو اساین کنیم به یه متغییر دیگه
بلکه این هست که میتونیم فانکشن های تو در تو داشته باشیم
یعنی یه فانکشن رو داخل یه فانکشن دیگه تعریف کنیم

خود این فیچر زمینه رو فراهم میکنه برای مفهوم بعدی
closure
که موضوع پست بعدی خواهد بود
